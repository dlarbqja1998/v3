# 2026-07-15 셔틀 기능 시안 개발 로그

## 작업 배경

- 메인 지도 화면에 셔틀 정보를 어떻게 노출할지 기획 논의 후 시안 구현을 진행했다.
- 셔틀은 학교와 조치원역 후편을 오가는 단일 코스 중심으로 보고, 메인 화면에는 요약 카드만 노출하기로 했다.
- 지도 핀은 광고처럼 기본 노출되는 방식이 아니라, 사용자가 보고 싶은 카테고리를 선택했을 때만 표시되도록 정리했다.

## 구현 내용

- 홈 바텀시트의 `다음 셔틀` 카드를 누르면 셔틀 전용 바텀시트가 열리도록 변경했다.
- 셔틀 바텀시트에서 다음 출발, 남은 시간, 출발 정류장 선택, 이후 출발 목록을 확인할 수 있게 했다.
- 셔틀 정류장 좌표를 실제 탑승 지점 기준으로 반영했다.
  - 학교 셔틀버스 타는 곳: `36.60948187479449, 127.28717680834059`
  - 조치원역 뒷편 셔틀버스 타는 곳: `36.600356245236036, 127.29520662494268`
- 셔틀 데이터는 `src/lib/domain/shuttle.ts`에 정적 시간표 데이터와 남은 시간 계산 함수로 분리했다.
- 셔틀 모드에서만 셔틀 정류장 핀이 표시되도록 지도 데이터 주입을 분리했다.
- 메인 기본 상태에서는 모든 핀이 보이지 않도록 변경했다.
- 사용자가 카테고리 필터를 선택하면 핀이 표시되고, 같은 카테고리를 다시 선택하면 필터가 해제되어 핀 없는 기본 화면으로 돌아가게 했다.
- `전체` 필터도 같은 방식으로 토글 해제되도록 처리했다.
- 지도 컴포넌트에서 셔틀 핀은 일반 핀과 색상 및 `BUS` 라벨로 구분되게 했다. 단, 이 핀은 셔틀 바텀시트가 열린 상태에서만 표시된다.

## 변경 파일

- `src/routes/+page.svelte`
  - 셔틀 바텀시트 모드 추가
  - 카테고리 필터 선택/해제 로직 추가
  - 기본 지도 핀 비노출 처리
- `src/lib/domain/shuttle.ts`
  - 셔틀 정류장, 시간표, 남은 시간 계산 유틸 추가
- `src/lib/map/NaverMap.svelte`
  - 셔틀 핀 시각 구분 처리

## 검증

- `npm run check` 통과
- 개발 서버 `http://localhost:5173` 응답 확인
- `npm run build`는 SSR/client 번들 생성까지 성공했으나, Cloudflare adapter 정리 단계에서 Windows 파일 잠금 오류가 반복 발생했다.
  - 오류: `EBUSY: resource busy or locked, rmdir 'C:\v3\.svelte-kit\cloudflare'`
  - 코드 컴파일 오류가 아니라 로컬 빌드 산출물 폴더 잠금 문제로 판단된다.

## 후속 작업

- 실제 셔틀 시간표를 확정 데이터로 교체해야 한다.
- 평일/주말/방학 운행 여부가 있다면 시간표 데이터 구조에 `dayType`을 확장해야 한다.
- 조치원역 후편과 학교 사이 이동 시간을 표시할지 추가 검토가 필요하다.
