# 골라바유 v3 신규 구축 목표

## 목표

골라바유 v3는 기존 맛집 목록 서비스가 아니라 고려대 세종 학생들이 매일 여는 로컬 생활 플랫폼으로 만든다. 첫 화면은 목록이 아니라 네이버 지도 기반 생활 홈이며, 식당, 학식, 셔틀버스, 모임, 리뷰, 공지성 생활 정보를 지도와 하단 시트 안에서 빠르게 확인할 수 있게 한다.

## 핵심 방향

1. 기존 v2는 운영 상태로 유지한다.
2. v3는 새 폴더, 새 Cloudflare 프로젝트, 새 Neon 프로젝트, 새 DB 스키마로 시작한다.
3. 기존 카카오 3초 로그인과 하이브리드 온보딩 철학은 유지하되, v3 환경에 맞춰 새로 연결한다.
4. 지도는 네이버 지도를 기본 제공자로 둔다.
5. 웹/PWA로 먼저 출시하고, 구조는 Capacitor 앱 래핑을 전제로 설계한다.
6. PostHog를 처음부터 붙여 사용자 행동 데이터를 쌓는다.
7. 일일 500명 규모를 1차 운영 기준으로 둔다.

## v3에서 반드시 해결할 것

1. 메인 진입 즉시 학생이 쓸 이유가 있어야 한다.
2. 지도에서 식당, 학식, 셔틀 정류장을 바로 볼 수 있어야 한다.
3. 지도 조작마다 외부 장소 API를 호출하지 않는다.
4. 장소, 구역, 카테고리, 셔틀, 학식 데이터를 우리 DB 중심으로 관리한다.
5. 기존 v2 운영과 v3 개발이 서로 영향을 주지 않아야 한다.
6. 나중에 앱으로 감쌀 때 로그인, 지도, PostHog, 딥링크, 푸시가 크게 흔들리지 않아야 한다.

## MVP 범위

1. 네이버 지도 기반 메인 홈
2. 구역 선택
3. 카테고리 필터
4. 식당 마커와 식당 하단 미리보기
5. 학식 진입 또는 학식 핀
6. 셔틀 정류장 핀과 다음 셔틀 시간
7. 카카오 로그인
8. 하이브리드 온보딩
9. 기본 모임/리뷰 흐름
10. PostHog 핵심 이벤트 수집

## MVP 이후

1. 앱 래핑
2. 푸시 알림
3. 딥링크
4. 관리자 장소 관리 도구
5. 장소 검증/동기화 도구
6. 추천 랭킹
7. 개인화 추천
